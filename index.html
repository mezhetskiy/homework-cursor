• git config --global user.name blabla - задаем свое имя пользователя, чтобы остальные видели, кто производил все операции в git'е
  • git config --global user.email blabal@bla - аналогично вышенаписанному, только задаем почту
  • git init - иницилизируем новый проект в git
  • git status - определяем (проверяем) состояние файлов в git
  • git add - подготавливаем файлы к отправке. Команду используем для всех untracked файлов
  • git commit - важнейшая команда в этом списке. С помощью нее фиксируем все изменения файлов (давая коммитам осмысленные названия). Позволяет вернуться, если надо, к предыдущим версиям файлов. Аналог слота сохранения в играх ;)
  • git log - отслеживаем все совершенные изменения. Можно чекнуть, кто, что и когда сделал :)
  • git checkout - возвращаем(откатываем) файлы до определенного состояния(коммита)
  • git branch - создаем новую ветку. git branch "название ветки"
  • git push - отправляем измененный файл на сервер (пушим). указываем имя репозитория и название ветки, куда отправляем файлы
  • git pull - получаем все измененные файлы на конкретной ветке конкретного репозитория. В команде дописываем название репо\ветки.
  • git merge - соединяем одну или несколько веток в текущую
  • git clone - клонируем репозиторий
  • git revert - создаем новый коммит и отменяем предыдущий ( не удаляя его )

    • .gitignore - определяет, файлы с какими расширениями не будут загружаться в репозиторий

  • git reset - команда для отмены каких-либо изменений
  • git rebase - переносим изменения из одной ветки в другую. Rebase сжимает все изменения в "патч" и интегрирует его в целевую ветку.
  • git cherry-pick - переносим ОТДЕЛЬНЫЕ коммиты из одного места в другое.
  • git stash - на время помещаем изменения в стэш, на сохранение. Изменения "пропадают" из рабочей директории. Можно восстановить их в любое время. Другими словами, откладывание :)
